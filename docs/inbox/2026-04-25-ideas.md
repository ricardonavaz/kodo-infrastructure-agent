### IDEA #3 — SQLite error "no such column: 'now'" al guardar playbook runs

**Capturada:** 2026-04-25 ~04:26

**Contexto:** Verificacion visual de arq-16. El streaming de eventos funciona correctamente, pero al final de cada playbook ejecutado aparece un error de SQLite. El error estaba latente (siempre fallaba el guardado del run en BD), pero no se veia porque arq-16 ocultaba todo el output.

**Texto original (de Ricardo, basado en screenshots):**

> "[error] no such column: 'now' - should this be a string literal in single-quotes?"

**Categorización automática:**

- Dominio: código + arquitectura (backend BD)
- Tipo: bug
- Urgencia: alta (afecta persistencia de runs en BD)
- Bloquea trabajo actual: no (arq-16 cerrado tecnicamente)
- Conexiones: arq-16, server/routes/playbooks.js, sistema de runs

**Mi interpretación inicial:**

En el flow de guardar un playbook run en SQLite (probablemente INSERT INTO playbook_runs ... VALUES (..., now, ...)), el codigo usa `now` sin comillas, que SQLite interpreta como nombre de columna en lugar de:
- string literal: 'now'
- funcion: datetime('now') o CURRENT_TIMESTAMP

Sin este fix, los runs probablemente NO se estan guardando en BD (o se guardan con timestamp incorrecto). Eso explicaria si la pestaña "Runs" muestra historico incompleto.

**Fix sugerido:**

1. Localizar el INSERT/UPDATE problematico:
   grep -rn '"now"\|`now`\|, now,' server/
   o
   grep -rn 'INSERT.*playbook_runs\|UPDATE.*playbook_runs' server/

2. Reemplazar con datetime('now') o CURRENT_TIMESTAMP segun mejor practica del schema actual.

3. Verificar que otros INSERTs similares en el codigo no tengan el mismo error.

**Prioridad:** Alta. Bug funcional que afecta persistencia. Atender en proximo fix.

**Estimado:** 30-60 minutos (depende de cuantos lugares tengan el mismo patron).

**Estado:** sin procesar

**Destino propuesto:** crear arq-19 al backlog en proximo procesamiento.
